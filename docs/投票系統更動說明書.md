# 🗳️ 投票系統更動說明書

適用範圍：日後需要大幅減少參加者、增加參加者、改人名、重排參加者時使用。

## 🧠 核心原理

目前重置 script 的資料來源順序如下：

1. `表格回應 5` 第 1 行，由 B 欄開始：參加者名單來源
2. `計算後台`：由 script 重建參加者名單、票數公式、加權分數
3. `大螢幕展示`：由 script 重建頭三名公式
4. `重置控制台`：只負責觸發重置

重要結論：

- ✅ 改 `表格回應 5` 第 1 行 header，重置會跟新名單。
- ✅ 改 Google Form 題目，只要同步到 `表格回應 5` 第 1 行，重置會跟新名單。
- ❌ 只改 `計算後台` 或 `大螢幕展示`，下次重置會被覆蓋。
- ❌ 只隱藏欄位不等於刪除參加者；script 仍可能讀到 header。

## ✅ 每次大改前必做

1. 📦 先備份整份 Google Sheet：`File` -> `Make a copy`
2. ⏸️ 暫停投票：Google Form -> `Responses` -> 關閉 accepting responses
3. 🔍 檢查 `表格回應 5` 第 1 行是否代表最新參加者名單
4. 🧪 改完後先交 1 份測試回應
5. 🔁 用 `重置控制台` 的 checkbox 執行一次重置
6. ✅ 檢查 `計算後台` 同 `大螢幕展示` 是否正確

## ➕ 增加參加者

建議流程：

1. 去 Google Form
2. 複製一條現有參加者題目
3. 改成新參加者名稱
4. 確保選項仍然是：
   - `自信`
   - `正常`
   - `待進`
5. 儲存 Form
6. 提交 1 份測試回應，令新欄位出現在 `表格回應 5`
7. 回到 Sheet，檢查 `表格回應 5` 第 1 行是否出現新參加者
8. 如要調整次序，直接調整 `表格回應 5` 的欄位順序
9. 去 `重置控制台` 勾選重置 checkbox
10. 檢查 `計算後台` 是否出現新參加者

判斷成功：

- ✅ `計算後台` A 欄有新參加者
- ✅ 新參加者有 `自信 / 正常 / 待進 / 加權` 公式
- ✅ `大螢幕展示` 頭三名仍正常

## ➖ 減少參加者

建議流程：

1. 去 Google Form
2. 刪除不再投票的參加者題目
3. 回到 `表格回應 5`
4. 檢查第 1 行是否仍保留舊參加者 header
5. 如果舊參加者 header 仍存在：
   - 先確認該參加者已不會再出現在 Form
   - 再刪除該參加者對應的整個欄位
6. 去 `重置控制台` 勾選重置 checkbox
7. 檢查 `計算後台` A 欄是否只剩有效參加者

判斷成功：

- ✅ 被刪除參加者不再出現在 `計算後台`
- ✅ 被刪除參加者不再影響頭三名
- ✅ `表格回應 5` 第 1 行只保留有效參加者

注意：

- ⚠️ 不要只在 `計算後台` 刪除參加者。
- ⚠️ 不要只 hide 欄位。
- ⚠️ 不要保留空白 header 夾在參加者中間。

## 🔄 大幅重整參加者名單

以下情況屬於大幅重整：

- 參加者增減超過三分之一
- 參加者次序大幅改動
- 分組改名
- 要重新設計 Form 題目格式
- 舊 response sheet 已經太亂

建議用乾淨流程：

1. 📦 複製舊 Sheet 做備份
2. 🧾 在 Google Form 內整理好全部參加者題目
3. 🧪 提交 1 份測試回應
4. 🔍 檢查 `表格回應 5` 第 1 行
5. 🧹 刪除不再需要的舊 participant 欄位
6. 🔁 執行 `重置控制台` checkbox
7. ✅ 檢查 `計算後台` 參加者數量是否等於 Form 題目數量

如果你決定建立全新的 response tab，例如 `表格回應 6`：

- 需要修改 Apps Script 內的 `RESPONSE_SHEET_NAME`
- 目前值是 `表格回應 5`
- 改完後要重新 Run `setupResetButton`

## 🧾 哪些改動需要改 Apps Script

不需要改 code：

- ✅ 只增加參加者
- ✅ 只減少參加者
- ✅ 改參加者名稱
- ✅ 改參加者排序
- ✅ 保持 response tab 仍叫 `表格回應 5`
- ✅ 保持選項仍是 `自信 / 正常 / 待進`

需要改 code：

- ⚠️ response tab 改名，例如由 `表格回應 5` 變成 `表格回應 6`
- ⚠️ 選項改名，例如 `自信` 改成 `好有自信`
- ⚠️ 權重改變，例如 `自信=5`
- ⚠️ 頭三名規則改變
- ⚠️ 想加新分類、分組排名、每組冠軍

## 🧪 更動後驗收清單

每次改完人數後，逐項檢查：

- [ ] Google Form 題目數量正確
- [ ] 每條參加者題目都有 `自信 / 正常 / 待進`
- [ ] `表格回應 5` 第 1 行 B 欄開始的 header 正確
- [ ] 沒有舊參加者 header 殘留
- [ ] 沒有空白 header 夾在參加者中間
- [ ] `重置控制台` checkbox 可勾選
- [ ] `計算後台` 參加者數量正確
- [ ] `計算後台` B:D 欄公式正常
- [ ] `計算後台` E 欄加權正常
- [ ] `大螢幕展示` 頭三名正常

## 🚫 常見錯誤

### ❌ 只改 `計算後台` 的名字

結果：下次重置會被 `表格回應 5` header 覆蓋。

正確做法：改 Google Form 題目，並確認 `表格回應 5` header 已同步。

### ❌ 刪除 Form 題目，但 Sheet header 還在

結果：重置後舊參加者仍會回到 `計算後台`。

正確做法：確認該題目不再收回應後，刪除 `表格回應 5` 對應欄位。

### ❌ 只 hide 欄位

結果：script 仍可能讀到 hidden 欄位 header。

正確做法：備份後刪除整欄，或清空 header。

### ❌ 改選項文字

結果：公式仍在數 `自信 / 正常 / 待進`，改名後票數會變 0。

正確做法：不要改選項文字；如一定要改，要同步修改 Apps Script 及公式。

## 🧯 回滾方法

如果改錯：

1. 開啟之前複製的備份 Sheet
2. 對照 `表格回應 5` 第 1 行
3. 把正確 participant header 複製回現有 Sheet
4. 執行 `重置控制台` checkbox
5. 檢查 `計算後台` 和 `大螢幕展示`

如果已經收了新投票：

- 不要直接刪 row
- 先複製整份 Sheet 備份
- 再決定是否清空或重置

## 📌 最短版口訣

`表格回應 5` 第 1 行 = 真正參加者名單。

改人名、加人、減人，都要令這一行正確。

然後按 `重置控制台` checkbox，系統就會重建後台公式。
